Данный класс (`TreeOfModel`) реализует **интерактивное дерево иерархии модели** для редактора или обучающего приложения на движке UNIGINE. Он создаёт виджет `WidgetTreeBox` внутри `ObjectGui`, отображая все узлы (Node) переданной модели в виде древовидной структуры. Класс предоставляет удобный интерфейс для навигации по составным объектам, управления их видимостью и получения быстрого доступа к отдельным деталям.

**Основные возможности:**

- **Построение дерева узлов:**  
  Рекурсивно обходит всех потомков корневой ноды (получаемой через `GetterModels`), присваивает каждому уникальный индекс и сохраняет в `WidgetTreeBox` с именем узла. Дерево отображается в левой части GUI.

- **Визуализация и подсветка:**  
  При клике на элемент дерева соответствующий объект (или вся ветвь) подсвечивается на 3D-сцене с помощью `Visualizer` (синий bounding box). Это помогает пользователю локализовать деталь в пространстве.

- **Управление видимостью (двойной клик):**  
  Двойной щелчок по элементу дерева переключает состояние детали:  
  – **Обычный режим:** деталь видна и участвует в пересечениях (исходный материал).  
  – **Скрытый режим:** деталь становится прозрачной (назначается полупрозрачный материал `transparent.mat`) и отключается от рейкастинга (`SetIntersection(false)`).  
  Состояние визуализируется изменением цвета элемента в дереве (красный – скрыт, белый – видим).

- **Запоминание исходных материалов:**  
  При первом построении дерева сохраняются оригинальные материалы всех поверхностей (`initialMaterials`), чтобы при возврате видимости можно было восстановить их, а не просто убрать прозрачность.

- **Множественный выбор:**  
  `WidgetTreeBox` настроен на мультивыбор (`MultiSelection = true`), что позволяет выделять несколько узлов одновременно (хотя в текущем коде обработка множественного выбора не реализована полностью, заложена основа).

- **Интеграция с `GetterModels`:**  
  Класс получает через конструктор экземпляр `GetterModels` и использует его для доступа к корневой ноде модели (`GetNode()` / `GetNodeID()`), что обеспечивает гибкость при смене целевой модели без жёсткой привязки к имени.

**Назначение в проекте:**  
`TreeOfModel` применяется в обучающих модулях, где необходимо детально изучать устройство сложных объектов: пользователь может скрывать части модели, чтобы видеть внутреннее строение, или быстро находить нужную деталь по её названию в иерархии. Это незаменимый инструмент для VR и ПК-тренажёров, разборки/сборки узлов.
